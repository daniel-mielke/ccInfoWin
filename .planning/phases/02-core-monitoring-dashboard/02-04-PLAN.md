    **1. SettingsViewModel.cs** -- New ViewModel for Settings page:

    DI dependencies: `ISettingsService`, `ICredentialService`, `INavigationService`

    **Refresh interval** -- ComboBox items model:
    ```csharp
    public record RefreshOption(string Label, int Seconds);

    public List<RefreshOption> RefreshOptions { get; } = new()
    {
        new("30 Sekunden", 30),
        new("1 Minute", 60),
        new("2 Minuten", 120),
        new("5 Minuten", 300),
        new("10 Minuten", 600),
        new("Manuell", 0)   // 0 means manual-only (no auto-refresh)
    };
    ```

    Observable properties:
    - `[ObservableProperty] private RefreshOption _selectedRefreshOption;` -- with partial method `OnSelectedRefreshOptionChanged` to save to settings and notify MainViewModel
    - `[ObservableProperty] private bool _isDarkMode;` -- with partial method `OnIsDarkModeChanged` to save and send ThemeChangedMessage

    **Initialize** (called on page load):
    ```csharp
    public void Initialize()
    {
        var settings = _settingsService.LoadSettings();
        _selectedRefreshOption = RefreshOptions.FirstOrDefault(o => o.Seconds == settings.RefreshIntervalSeconds)
                                 ?? RefreshOptions[1]; // default 60s
        _isDarkMode = settings.ColorMode != "light"; // default dark
        // Raise PropertyChanged for initial binding
        OnPropertyChanged(nameof(SelectedRefreshOption));
        OnPropertyChanged(nameof(IsDarkMode));
    }
    ```

    **OnSelectedRefreshOptionChanged** partial method:
    ```csharp
    partial void OnSelectedRefreshOptionChanged(RefreshOption value)
    {
        var settings = _settingsService.LoadSettings();
        settings.RefreshIntervalSeconds = value.Seconds;
        _settingsService.SaveSettings(settings);
        // MainViewModel will pick this up when navigating back (or via messenger)
    }
    ```

    **OnIsDarkModeChanged** partial method:
    ```csharp
    partial void OnIsDarkModeChanged(bool value)
    {
        var colorMode = value ? "dark" : "light";
        var settings = _settingsService.LoadSettings();
        settings.ColorMode = colorMode;
        _settingsService.SaveSettings(settings);
        WeakReferenceMessenger.Default.Send(new ThemeChangedMessage(colorMode));
    }
    ```

    **Commands:**
    - `[RelayCommand] private void Logout()` -- clear credentials, send AuthStateChangedMessage(false), navigate to LoginView (same logic as current MainViewModel.Logout)
    - `[RelayCommand] private void GoBack()` -- `_navigationService.GoBack()`

    **MainWindow.xaml.cs** -- Handle ThemeChangedMessage to apply theme changes immediately:

    1. Implement `IRecipient<ThemeChangedMessage>`:
    ```csharp
    public sealed partial class MainWindow : Window, IRecipient<ThemeChangedMessage>
    ```

    2. Register in constructor (after existing code):
    ```csharp
    WeakReferenceMessenger.Default.Register<ThemeChangedMessage>(this);
    ```

    3. Implement Receive:
    ```csharp
    public void Receive(ThemeChangedMessage message)
    {
        if (Content is FrameworkElement fe)
        {
            fe.RequestedTheme = message.Value == "light"
                ? ElementTheme.Light
                : ElementTheme.Dark;
        }
    }
    ```

    4. Also apply persisted theme on window creation (may already be handled in App.xaml.cs from Plan 01, but verify):
    - Read from MainWindow directly if App.xaml.cs timing is problematic. The most reliable place is after `Content = rootFrame` in MainWindow constructor. Load settings there and apply theme.

    NOTE: Do NOT use `Application.RequestedTheme` -- it throws at runtime in WinUI 3. Always use `FrameworkElement.RequestedTheme` on the window's Content element.
